# Sound Sources — Phase 8.1

All ambient tracks are licensed CC0 field recordings from Freesound.org.
Each output file is encoded MP3 96kbps · 44.1kHz · mono · ~60s with
loudnorm `I=-23 TP=-2 LRA=7`, produced by `scripts/process-sounds.mjs`.
Source files are public Freesound HQ preview MP3s (no login required);
they are NOT AI-generated and NOT procedurally generated.

**No boundary fade in/out is baked into the files.** Every track is
intended to native-loop (`audio.loop=true`) seamlessly; baked-in fades
caused an audible ~1s silence dip at every loop point. The TP=-2 dBTP
ceiling from loudnorm provides enough headroom to avoid clicks at the
wraparound.

Downloaded date: **2026-05-01**

## air-purifier.mp3
- Source: pushkin — "01 room tone low frequency hvac.flac"
- URL: https://freesound.org/people/pushkin/sounds/215293/
- Preview MP3: https://cdn.freesound.org/previews/215/215293_336074-lq.mp3
- License: CC0 (Creative Commons 0)
- Original duration: 0:34
- Downloaded date: 2026-05-02
- Trimmed to: 60s (stream-loop of clean inner segment 1.0s–32.0s → ~60s)
- Notes: Low-level, low-frequency fan noise, very quiet. Author created
  the recording to fill dictation gaps; no clicks/interruptions in the
  source. Stereo source is converted to mono by the pipeline (`pan=mono`).
  User-selected for quieter/cleaner low-level fan character vs the
  previous jbeetle AC loop. Used as free "공기청정기" track.
- Loop seam: the source recording itself ships with a baked fade-in
  (~0–1s) and fade-out (~33–34s). Stream-looping the raw source leaks
  those into the 60s output, leaving a perceptible level dip at the
  file's loop boundary. The pipeline therefore extracts the steady body
  (`start=1.0s`, `srcEnd=32.0s`) as an intermediate WAV first, then
  stream-loops *that* clean segment to 60s. Result: file boundaries
  match (first 0.5s = -24.9 dB, last 0.5s = -24.3 dB; mid = -24.0 dB) —
  no fade audible at `audio.loop=true` wraps.
- Per-source gain: **-4 dB** trim applied after loudnorm
  (`volume=-4dB`). Low-frequency-dominated content reads at `-23 LUFS`
  on the integrated meter but is perceptually louder than the other
  tracks at the same shared app volume; user reported it as too loud.
